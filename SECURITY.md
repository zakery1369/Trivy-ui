# Security Policy

Trivy UI is a free and open-source project published publicly on GitHub.

Trivy UI is a web-based user interface for Trivy and may be used to display, analyze, and review security scan results. However, Trivy UI is provided without any warranty, support commitment, or security responsibility.

Use of this project is entirely at the user's own risk.

This project is an independent web UI for Trivy and is not affiliated with, endorsed by, or sponsored by Aqua Security or the official Trivy project.

The name "Trivy" belongs to its respective owners and is used here only to describe compatibility and integration with Trivy.

## No Security Warranty

Trivy UI is provided "as is".

The project maintainer does not guarantee:

- That the project is fully secure
- That the project is suitable for production environments
- That the project is free from vulnerabilities
- That the analysis or outputs are fully accurate
- That the project is compatible with all versions of Trivy
- That reported issues will be fixed quickly or at all
- That old or future versions will be continuously supported

Users should perform their own security reviews and testing before using Trivy UI in real, organizational, production, or sensitive environments.

## Supported Versions

Trivy UI does not currently provide official security support for any version.

| Version | Official Security Support |
| ------- | ------------------------- |
| main branch | No |
| releases | No |

Some security issues may be reviewed and fixed when possible, but there is no commitment to respond, patch, release fixes, or provide security support.

## Reporting a Vulnerability

If you find a security vulnerability, you may report it through GitHub Issues or through the available contact methods of the project maintainer.

Please avoid sharing sensitive information in public reports.

Sensitive information includes:

- API keys
- Access tokens
- Passwords
- Secrets
- Private keys
- Private URLs
- Internal IP addresses
- Scan data from private projects
- Full vulnerability reports from personal or organizational projects
- Full SBOM outputs from private projects
- Customer or company data
- Private container image names
- Internal repository names

## Project Scope

Trivy UI displays and processes results generated by Trivy. Vulnerabilities found by Trivy in packages, containers, images, dependencies, operating system packages, or files from other projects are not necessarily vulnerabilities in Trivy UI itself.

Issues that may be related to Trivy UI include:

- Unsafe rendering or processing of Trivy output
- Unintended exposure of scan data
- Unsafe rendering of report content
- Issues in the project API or backend
- Issues in the Dockerfile or runtime configuration
- Issues related to AI features
- Unsafe storage or display of sensitive data
- Incorrect handling of imported scan reports
- UI or backend behavior that could expose private scan results

## AI Features

Trivy UI may include AI-powered features for explaining vulnerabilities, summarizing findings, or suggesting remediation steps.

AI-generated output is provided for assistance only. It should not be considered a definitive result, official security advice, or a replacement for expert review.

Users are responsible for reviewing, validating, and making final decisions based on any AI-generated suggestions.

The project maintainer does not guarantee the accuracy, safety, completeness, or reliability of AI-generated content.

Users should also review what data may be sent to any AI provider before enabling or using AI-related features.

## Use in Sensitive Environments

Trivy UI should be used with caution in production environments, organizational infrastructure, sensitive systems, or projects containing confidential data.

Before using Trivy UI in such environments, users should review and validate:

- Security configuration
- Access control
- Data storage behavior
- Data sent to AI services
- Dependency status
- Docker and deployment configuration
- User access to scan results
- Network exposure
- Authentication and authorization settings
- Handling of uploaded or imported scan reports
