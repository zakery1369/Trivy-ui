# Trivy Docker Image Scanner UI

[فارسی](README.md) | [English](README.en.md)

[راهنمای مشارکت](CONTRIBUTING.md) | [قالب Pull Request](.github/PULL_REQUEST_TEMPLATE.md)

![Trivy UI](https://raw.githubusercontent.com/zakery1369/pics/refs/heads/master/Trivy-UI.png)

یک رابط کاربری فارسی، ساده و مدرن برای اسکن امنیتی Docker Imageها با استفاده از [Trivy](https://github.com/aquasecurity/trivy).

این پروژه کمک می‌کند بدون نیاز به اجرای مستقیم دستورات Trivy، Docker Imageها را اسکن کرده، آسیب‌پذیری‌ها را مشاهده کنید، گزارش امنیتی دریافت کنید و با استفاده از هوش مصنوعی، تحلیل و پیشنهادهای اصلاحی دریافت کنید.

## ✨ قابلیت‌ها

## 🔍 اسکن امنیتی Docker Image

- رابط کاربری دوزبانه فارسی (RTL) و انگلیسی (LTR)
- نمایش نسخه Trivy در داشبورد
- مشاهده Docker Imageهای موجود روی سیستم
- امکان اسکن Imageهای Remote
- Pull خودکار Image در صورت عدم وجود روی سیستم
- مدیریت دستی دیتابیس Trivy
- عدم آپدیت خودکار دیتابیس در هر اسکن برای افزایش سرعت

نمونه Image:

```bash
nginx:latest
alpine:3.19
python:3.12-slim
ubuntu:22.04
```

## 📊 نمایش و تحلیل نتایج اسکن

نمایش آسیب‌پذیری‌ها بر اساس سطح شدت:

- Critical
- High
- Medium
- Low
- Unknown


امکانات گزارش:

- فیلتر نتایج بر اساس Severity
- جستجو در نتایج اسکن
- نمایش خلاصه آسیب‌پذیری‌ها
- دریافت گزارش در فرمت‌های مختلف:

  - HTML
  - JSON
  - SARIF
  - TXT


---

## 🌐 رابط کاربری دوزبانه (فارسی و انگلیسی)

Trivy UI به‌صورت داخلی از **فارسی** و **English** پشتیبانی می‌کند. با دکمه تغییر زبان در بالای داشبورد می‌توانید زبان را فوراً و بدون بارگذاری مجدد صفحه عوض کنید.

- ترجمه متن‌های داشبورد، دکمه‌ها، وضعیت‌ها، فیلترها، placeholderها، tooltipها و برچسب‌های دسترس‌پذیری
- تغییر خودکار زبان و جهت صفحه: **راست‌به‌چپ (RTL)** برای فارسی و **چپ‌به‌راست (LTR)** برای انگلیسی
- نمایش اعداد متناسب با زبان انتخاب‌شده (`fa-IR` یا `en-US`)
- ذخیره زبان انتخابی در `localStorage` مرورگر و حفظ آن در مراجعه بعدی
- بازسازی خلاصه اسکن، جدول آسیب‌پذیری‌ها و پیام‌های پویا پس از تغییر زبان
- ارسال زبان انتخابی همراه درخواست تحلیل AI تا پیشنهادهای اصلاحی به همان زبان رابط کاربری تولید شوند
- استفاده از فارسی به‌عنوان زبان پیش‌فرض و fallback در صورت نبود تنظیم ذخیره‌شده یا ترجمه

تغییر زبان فقط نحوه نمایش رابط و زبان پاسخ AI را تغییر می‌دهد. شناسه‌های CVE، نام پکیج‌ها، نسخه‌ها، آدرس Imageها و سایر مقادیر فنی برای حفظ دقت بدون تغییر باقی می‌مانند.

# 🤖 تحلیل هوشمند گزارش Trivy با AI

یکی از قابلیت‌های اصلی پروژه، امکان تحلیل گزارش‌های امنیتی Trivy با استفاده از مدل‌های هوش مصنوعی است.

کاربر می‌تواند API Key سرویس هوش مصنوعی موردنظر خود را وارد کند تا بعد از پایان اسکن، گزارش Trivy توسط AI بررسی شده و پیشنهادهای امنیتی دریافت کند.


## قابلیت‌های تحلیل AI

هوش مصنوعی می‌تواند:

- آسیب‌پذیری‌های مهم را اولویت‌بندی کند
- سطح ریسک هر مورد را توضیح دهد
- علت ایجاد آسیب‌پذیری را بررسی کند
- پیشنهادهای اصلاحی ارائه دهد
- راهکارهای کاهش ریسک را پیشنهاد کند
- تغییرات موردنیاز در Docker Image یا Dependencyها را پیشنهاد دهد
- گزارش امنیتی قابل فهم برای تیم DevOps و Security تولید کند


> تحلیل AI جایگزین Trivy نیست؛ بلکه یک لایه هوشمند برای درک بهتر خروجی‌ها و تصمیم‌گیری سریع‌تر روی نتایج اسکن است.

# 📖 نحوه استفاده


## 1. انتخاب Docker Image

در صفحه اصلی، لیست Docker Imageهای موجود روی سیستم نمایش داده می‌شود.

کاربر می‌تواند Image موردنظر را انتخاب کرده و فرآیند اسکن را شروع کند.


---

## 2. اسکن Imageهای Remote

در صورتی که Image موردنظر روی سیستم وجود نداشته باشد، می‌توانید نام Image را به‌صورت مستقیم وارد کنید.

مثال:

```bash
nginx:latest

ubuntu:22.04

redis:7

python:3.12-slim
```


در این حالت برنامه:

1. بررسی می‌کند Image روی سیستم وجود دارد یا خیر
2. در صورت نیاز آن را Pull می‌کند
3. اسکن امنیتی را با Trivy انجام می‌دهد


---

## 3. به‌روزرسانی دیتابیس Trivy

برای افزایش سرعت اجرای Scan، برنامه به‌صورت پیش‌فرض قبل از هر اسکن دیتابیس Trivy را Update نمی‌کند.


کاربر می‌تواند از طریق داشبورد، دیتابیس Trivy را به‌صورت دستی به‌روزرسانی کند.


مزایا:

- کنترل بهتر روی زمان Update
- کاهش زمان اجرای Scan
- جلوگیری از دانلودهای تکراری


---

## 4. تحلیل گزارش با هوش مصنوعی

بعد از پایان اسکن، کاربر می‌تواند گزارش Trivy را برای تحلیل AI ارسال کند.


مراحل:

1. انجام Scan توسط Trivy
2. تولید گزارش آسیب‌پذیری‌ها
3. ارسال گزارش به مدل AI انتخاب‌شده
4. دریافت تحلیل امنیتی و پیشنهادهای اصلاحی


خروجی AI شامل مواردی مانند:

- مهم‌ترین آسیب‌پذیری‌ها
- اولویت رفع مشکلات
- پیشنهاد تغییرات لازم
- راهکارهای کاهش ریسک

# ⚠️ نکته امنیتی مهم

این پروژه برای دسترسی به Docker Imageهای سیستم، Docker Socket را داخل کانتینر Mount می‌کند:


```yaml
- /var/run/docker.sock:/var/run/docker.sock
```


Docker Socket دسترسی بسیار قدرتمندی دارد و در محیط‌های حساس باید با احتیاط استفاده شود.


پیشنهاد می‌شود این ابزار در محیط‌های کنترل‌شده اجرا شود:

- سیستم شخصی
- محیط تست
- شبکه داخلی
- سرورهای DevOps
- Security Lab


از اجرای این سرویس روی سرورهای عمومی بدون محدودیت دسترسی مناسب خودداری کنید.

# درباره Trivy

[Trivy](https://github.com/aquasecurity/trivy) یک ابزار متن‌باز امنیتی است که توسط Aqua Security توسعه داده شده است.


Trivy قابلیت بررسی موارد زیر را دارد:

- Docker Image Vulnerability Scanning
- OS Package Vulnerabilities
- Library Vulnerabilities
- File System Scanning
- Git Repository Scanning
- Kubernetes Security Scanning
- Infrastructure as Code (IaC) Misconfiguration Detection
